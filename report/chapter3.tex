\subsection{Структура программного решения}

Программная реализация веб-приложения для частной медицинской клиники выполнена на объектно-ориентированном языке программирования Python с использованием микрофреймворка Flask. Выбор Flask обусловлен его легковесностью, модульностью и простотой интеграции со сторонними библиотеками для работы с базами данных (Flask-SQLAlchemy) и организации сессий пользователей (Flask-Login).

Проект имеет следующую структуру директорий и файлов:
\begin{enumerate}
    \item \texttt{app.py} - основной файл приложения, содержащий инициализацию Flask-сервера, настройку обработчиков маршрутов (endpoints), логику обработки запросов и авторизации;
    \item \texttt{models.py} - модуль описания структуры базы данных на уровне объектно-реляционного отображения (ORM SQLAlchemy);
    \item \texttt{config.py} - конфигурационный класс приложения, определяющий настройки безопасности (секретный ключ сессии), путь к базе данных SQLite/PostgreSQL, параметры загрузки документов и лимиты на размер файлов;
    \item \texttt{seed.py} - скрипт для первоначального создания таблиц базы данных и наполнения их демонстрационными данными (учетными записями тестовых пользователей различных ролей, списком услуг и расписанием врачей);
    \item \texttt{requirements.txt} - перечень внешних зависимостей проекта и версий библиотек для сборки окружения;
    \item \texttt{templates/} - каталог HTML-шаблонов страниц с разметкой на базе шаблонизатора Jinja2;
    \item \texttt{static/} - папка для статических ресурсов приложения, включающая стили CSS, скрипты JavaScript, иконки Bootstrap Icons и подкаталог \texttt{uploads/} для загрузки медицинских заключений и файлов пациентов.
\end{enumerate}

Конфигурационный файл \texttt{config.py} обеспечивает адаптивность приложения к среде развертывания: при наличии в переменных окружения строки подключения \texttt{DATABASE\_URL} приложение подключается к внешней СУБД PostgreSQL (промышленный режим), иначе используется локальный файл базы данных \texttt{clinic.db} через встроенную поддержку SQLite (режим разработки).

\subsection{Модели данных и ORM-маппинг}

Для управления персистентным состоянием и выполнения запросов к СУБД в приложении применена технология ORM (Object-Relational Mapping) на базе Flask-SQLAlchemy. Описание логических сущностей и связей выполнено в виде Python-классов, наследующих базовый класс декларативного описания \texttt{db.Model}.

Основные классы моделей, реализованные в модуле \texttt{models.py}:
\begin{enumerate}
    \item \texttt{User} - учётная запись пользователя. Содержит поля \texttt{id}, \texttt{email}, \texttt{password\_hash} (хэш пароля на базе алгоритма scrypt) и \texttt{role} (роль: \texttt{'patient'}, \texttt{'doctor'} или \texttt{'admin'}). Реализует методы \texttt{set\_password} для хэширования и \texttt{check\_password} для проверки введённого пароля.
    \item \texttt{Patient} - профиль пациента. Связан отношением 1:1 с учётной записью \texttt{User} с использованием каскадного удаления \texttt{cascade="all, delete-orphan"}. Содержит персональные данные (ФИО, дата рождения, телефон).
    \item \texttt{Doctor} - профиль врача. Связан отношением 1:1 с учётной записью \texttt{User}. Дополнительно содержит поля специализации (ссылка на \texttt{Specialization}) и номера кабинета приёма \texttt{cabinet}.
    \item \texttt{Specialization} - справочник врачебных специальностей (например, терапевт, кардиолог, невролог).
    \item \texttt{Service} - перечень предоставляемых услуг с указанием названия, описания, цены и длительности в минутах.
    \item \texttt{ScheduleSlot} - интервалы расписания приёма. Содержит ссылку на \texttt{Doctor}, дату, время начала и окончания интервала, а также статус занятости слота (\texttt{'free'}, \texttt{'reserved'}).
    \item \texttt{Appointment} - запись на приём. Связывает пациента (\texttt{Patient}), врача (\texttt{Doctor}), услугу (\texttt{Service}) и временной интервал расписания (\texttt{ScheduleSlot}). Поле \texttt{slot\_id} снабжено ограничением уникальности \texttt{unique=True}, предотвращающим повторное бронирование одного слота.
    \item \texttt{MedicalRecord} - электронная медицинская карта. Хранит группу крови, резус-фактор, перечень аллергенов и хронических болезней пациента. Связана 1:1 с пациентом.
    \item \texttt{Visit} - протокол осмотра (посещения). Содержит жалобы, диагноз и рекомендации врача. Создаётся для конкретной записи \texttt{Appointment} после выполнения приёма.
    \item \texttt{Document} - прикрепленные файлы исследований (картинки, документы PDF). Связан с пациентом и (опционально) с конкретным посещением.
\end{enumerate}

Связи между таблицами настроены с использованием внешних ключей (\texttt{db.ForeignKey}) с поддержкой ограничений целостности БД (\texttt{ondelete='CASCADE'} и \texttt{ondelete='SET NULL'}), что обеспечивает корректную работу транзакций СУБД.

\subsection{Логика маршрутизации и бизнес-логика}

Маршрутизация запросов и бизнес-логика реализованы в виде контроллеров Flask в файле \texttt{app.py}. Доступ к защищённым страницам регулируется расширением \texttt{Flask-Login}. Ограничение доступа к страницам личных кабинетов реализовано с помощью декоратора \texttt{@login\_required} и проверок роли текущего авторизованного пользователя \texttt{current\_user.role}.

Основные функциональные маршруты приложения:
\begin{enumerate}
    \item \textbf{Публичные маршруты}:
    \begin{itemize}
        \item \texttt{/} (функция \texttt{index}) - отображение главной страницы с направлениями работы и списком услуг клиники;
        \item \texttt{/auth} (функция \texttt{auth\_page}) - страница аутентификации и регистрации;
        \item \texttt{/login} и \texttt{/register} (POST-обработчики) - выполняют валидацию входных данных, проверку уникальности email, авторизацию сессии с помощью \texttt{login\_user} и перенаправление пользователя в личный кабинет в зависимости от его роли.
    \end{itemize}

    \item \textbf{Личный кабинет пациента}:
    \begin{itemize}
        \item \texttt{/cabinet} (функция \texttt{patient\_cabinet}) - собирает информацию о профиле пациента, его электронной медицинской карте, а также списки будущих записей на приём и архив прошедших визитов с рекомендациями врачей;
        \item \texttt{/booking} (функция \texttt{booking\_page}) - страница онлайн-записи на приём. Передаёт в шаблон списки направлений, врачей и услуг;
        \item \texttt{/booking/get\_slots} - API-маршрут (возвращает JSON), используемый динамическим JavaScript на клиенте для получения свободных слотов расписания конкретного врача на выбранную дату;
        \item \texttt{/booking/create} (POST-запрос) - создание записи на приём. Бронирует выбранный слот (переводит его статус в \texttt{'reserved'}), создаёт объект \texttt{Appointment} и перенаправляет в кабинет.
    \end{itemize}

    \item \textbf{Личный кабинет врача}:
    \begin{itemize}
        \item \texttt{/doctor} (функция \texttt{doctor\_cabinet}) - отображает расписание приёмов врача на выбранную дату (по умолчанию на текущий день), позволяя фильтровать записи;
        \item \texttt{/doctor/visit/<int:appt\_id>} - страница проведения приёма пациента. Врач вносит текстовые жалобы, диагноз и рекомендации. По завершении создаётся запись \texttt{Visit}, статус приёма обновляется на \texttt{'completed'}, а слот расписания закрывается;
        \item \texttt{/doctor/upload\_document} - обработчик загрузки медицинских файлов. Выполняет проверку расширения файла, сохраняет его в папку \texttt{static/uploads} с безопасным именем и связывает запись \texttt{Document} с пациентом.
    \end{itemize}

    \item \textbf{Панель администратора}:
    \begin{itemize}
        \item \texttt{/admin} (функция \texttt{admin\_dashboard}) - сводная панель со статистикой работы клиники;
        \item \texttt{/admin/doctors} и \texttt{/admin/services} - управление справочниками медицинского персонала и услуг (добавление, редактирование, удаление записей);
        \item \texttt{/admin/appointments} - общий реестр записей на приём с возможностью подтверждения (\texttt{confirm}) или отмены (\texttt{cancel}).
    \end{itemize}
\end{enumerate}

При обработке форм реализован механизм всплывающих уведомлений \texttt{flash()}, информирующий пользователя об успешном завершении операции или возникших ошибках валидации данных.
